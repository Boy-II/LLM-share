### **專案目標**
本專案旨在定義一套完整、高效的流程，用於指導大型語言模型 (LLM) 生成一個**可配置的、帶有簡易主題編輯器的一頁式網站**。

### **核心理念**
*   **資產驅動 (Asset-Driven)**：流程的核心是預先建立或動態生成一系列可重用的「資產」（HTML 範本、CSS 配色、JS 功能），然後在生成階段進行「組裝」，而非從零創作。
*   **混合式指令 (Hybrid Prompting)**：結合「自然語言指令」（用於宏觀佈局和複雜邏輯）與「自定義簡寫語法」（用於高效處理重複性內容）的優點。
*   **互動式工作流 (Interactive Workflow)**：LLM 應作為一個「設計助理」，在遇到指令模糊或資訊缺失（佔位符）時，能主動暫停並向使用者詢問，以確保結果的準確性。

### **關鍵策略與創新**
1.  **三大資產庫**：
    *   `template_samples.md`：HTML 結構範本。
    *   `color_schemes.md`：CSS 配色方案。
    *   `js_templates.md`：可重用的 JavaScript 功能模組。
2.  **動態資產生成**：
    *   **色彩**：可透過分析使用者提供的參考網站截圖（純視覺分析，防止程式碼汙染）來動態提取配色方案。
    *   **特效**：可透過分析使用者提供的動畫影片（需優化時長與幀率）來重現動畫效果。
3.  **最終產出：網站 + 主題編輯器**：
    *   最終產出物不僅是網站本身，還包含一個 `theme-editor.html`。
    *   此編輯器允許使用者透過 UI 控制項，即時調整網站的 CSS 變數（顏色、字體等）。
    *   支援透過 `FileReader` API 實現無後端的圖片上傳與更換。
    *   所有客製化選項都將儲存在瀏覽器內的 `localStorage` 中，以實現狀態持久化。

### **待討論議題 (Next Topics)**

*   **響應式設計策略**：如何系統性地在 Prompt 中定義和要求不同螢幕尺寸（斷點）下的佈局變化？
*   **專案匯出與部署**：如何將使用者透過主題編輯器配置好的網站（包含 `localStorage` 中的設定）匯出為一套可獨立部署的靜態檔案？
*   **錯誤處理與恢復機制**：當 LLM 生成的程式碼出現錯誤或不符合預期時，應採取何種恢復或重試策略？
